Talking point — "Why an agent? I could just prompt Copilot"

True — you could type this as a one-off prompt. The point of an agent is that you do it once and keep it:

  • Reusable — one click every morning, no re-typing the prompt.
  • Consistent — the format is baked in (always [Internal]/[External], always "Top 3 actions").
  • Grounded & scoped — you set its knowledge and capabilities once.
  • Shareable & governed — hand it to the whole leadership team; IT can govern it.
A prompt is a one-off. An agent is that prompt turned into a reusable, shareable, governed capability.

Background — how agents work & where Agent Builder fits (30 sec)

A Copilot agent is just Copilot itself, specialized: the same model and orchestrator, plus your instructions, knowledge, and optional actions. That's a declarative agent — and Agent Builder is the no-code way to make one.

Anatomy of an agent

  • Instructions — how it behaves.
  • Knowledge — what it can read (email, Teams, SharePoint, files, web).
  • Actions — what it can do in other systems (APIs / connectors).
  • Orchestrator + model — Copilot's own engine; nothing to host.

Reference — what Agent Builder can do (explain on the go)

Knowledge sources (what it can ground on)

  • Outlook email — "My emails"  (needs Copilot add-on license)
  • Microsoft Teams — "My Teams chats and meetings", or scope to up to 5 specific chats / channels / meeting chats  (needs Copilot license)
  • SharePoint — sites, folders, files (up to 100 files); respects permissions & sensitivity labels
  • OneDrive — up to 50 files
  • Public websites — up to 4 URLs (max 2 levels, no query params), or a "Search all websites" toggle
  • Uploaded files — embedded from your device (Configure tab)
  • Copilot connectors — enterprise data (Graph connectors), if your admin enabled them

Up to 20 knowledge sources per agent. For this agent we add My emails + My Teams chats and meetings.

Capabilities (what it can do)

  • Web search — ground answers on live public web content (the [External] signals).
  • Code interpreter — runs Python to analyze data and build charts/tables; e.g. a quick table of open actions by urgency, or analyze an attached Excel.
  • Image generator — creates images from a prompt (DALL·E). Not needed for this agent.

Three ways to start

  • Describe (recommended) — natural language builds it for you.
  • Configure — fill the fields manually (Skip to configure).
  • Template — start from a preconfigured use case.

Agent 2 — what they'll find (your cheat sheet)

The agent computes this live — these are the planted stories so you know where it lands:

  • Air freight over-indexes hard — ~15% of shipments but ~34% of total CO₂e. The headline.
  • Sea is the biggest absolute emitter (~55%) because it carries the most tonnage — context, not villain.
  • Road is ~31% of shipments but only ~3–4% of CO₂ (short legs) → modal-shift to Rail story.
  • Sustainable fuel (SAF/biofuel) adoption rises through the year and visibly lowers emission intensity.
  • "Aurelia Pharma" shows up as an air-freight emission hotspot customer.
  • Total footprint ≈ 150,000 t CO₂e across 25,000 shipments.
